ITIL Standard processes

Four ITIL Core processes

ITSM Modules follow ITIL best practice design, covering core enterprise IT service management process requirements.

Incident Management

Rapid service restoration, minimizing business disruption impact.

  • Auto/manual ticket creation, alerts directly linked
  • P1-P4 Priority classification,SLA timer auto-starts
  • Topology root cause analysis: multiple device failures merged into one ticket

Problem Management

Deep analysis of recurring incidents to find and eliminate root causes.

  • Event correlation analysis, identifying recurring failure patterns
  • Root cause analysis and solution tracking
  • Knowledge base capture, preventing recurrence of similar issues

Change Management

Standardized change processes, reducing change risks, ensuring all changes are traceable.

  • Change request approval process, CI changes auto-trigger CR
  • Change impact assessment, based on CMDB topology analysis
  • Change implementation tracking and rollback plans

Service Request

Standardized handling of daily user service requests, improving service efficiency.

  • Self-service portal, users submit requests independently
  • Predefined service catalog and request templates
  • Real-time request progress tracking, service quality evaluation
