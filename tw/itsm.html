ITIL 標準流程

四大 ITIL 核心流程

ITSM 模塊遵循 ITIL 最佳實踐設計，覆蓋企業 IT 服務管理的核心流程需求。

事件管理 (Incident)

快速恢復服務，最小化業務中斷影響。

  • 自動/手動創建工單，告警直接關聯
  • P1-P4 優先級分級，SLA 計時器自動啓動
  • 拓撲根因分析：多臺設備故障合併爲一張工單

問題管理 (Problem)

深度分析反覆出現的事件，找到並消除根本原因。

  • 事件關聯分析，識別反覆出現的故障模式
  • 根因分析與解決方案追蹤
  • 知識庫沉澱，防止同類問題再次發生

變更管理 (Change)

標準化變更流程，降低變更風險，確保所有變更有跡可循。

  • 變更請求審批流程，CI 變更自動觸發 CR
  • 變更影響評估，基於 CMDB 拓撲分析
  • 變更實施追蹤與回滾計劃

服務請求 (Service Request)

標準化處理用戶日常服務請求，提升服務效率。

  • 自助服務門戶，用戶自主提交請求
  • 預定義服務目錄和請求模板
  • 請求進度實時查看，服務質量評價
智能告警

智能工單與告警儀表盤

從告警到工單的全自動化閉環，三種工單模式滿足不同運營需求。

推薦

全自動模式

基於 Poller 邏輯自動創建和分配工單。

  • 自動創建工單並分配
  • 利用網絡拓撲根因分析
  • 多設備故障自動關聯合並

半自動模式

控制分配到已有工單，限制新建工單數量。

  • 告警自動關聯已有工單
  • 限制重複創建
  • 人工確認後升級處理

手動模式

僅生成告警，用戶確認後才創建工單。

  • 告警推送但不自動創建工單
  • 人工評估後決定是否開單
  • 適用於非關鍵業務場景

告警儀表盤

跨客戶告警可視化，團隊協作透明高效

告警確認

團隊成員認領告警，狀態透明可見

創建工單

直接從告警創建工單或關聯已有工單

跨客戶視圖

SLA 管控

SLA 服務等級管理

基於工單優先級的 SLA 計時器，支持時區感知與自定義覆蓋時段，確保服務承諾精準達成。

優先級 說明 響應時間 緊急度
P1 最嚴重問題，對業務運營產生負面影響且無法通過可接受方式恢復 10 min 緊急
P2 次嚴重問題，對業務有較大影響但網絡仍可運行 20 min 高
P3 一般問題，網絡功能降低但對業務影響較小 30 min 中
P4 服務請求或計劃性工作 30 min 低
響應與解決分離

響應計時器和解決計時器分別配置，精確追蹤服務各環節效率。

時區感知

自動匹配工作時區，自定義工作時間/非工作時間覆蓋時段。

SLA 超時預警

SLA 即將超時自動升級告警等級，確保關鍵問題及時處理。

通知引擎

高度可定製的郵件/短信通知模板，HTML 編輯器支持，事件驅動精準推送。

  • 多語言、多時區感知 — 自動匹配收件人時區和語言
  • 事件驅動觸發 — 工單創建/更新/日誌創建/SLA 超時
  • HTML 模板編輯器 — 自定義郵件和短信通知格式
  • 第三方集成 — HTTP 調用外部系統，自定義請求頭和認證

工單模板與流程

預定義工單模板，支持 HTML 編輯器自定義輸入表單，完整的工單生命週期管理。

  • 工單日誌 (Journals) — 私密日誌、文件上傳、草稿
  • 聯繫人分配 — 自動/手動，區分主要/次要 (Email/CC)
  • 關聯工單 — 優化告警路由，避免重複處理
  • 自動關閉緩衝期 — 解決後允許客戶在最終關閉前介入

與 CMDB 深度聯動

開放生態

開放式生態對接

ITSM 模塊絕非孤立系統。ITHub 提供高開放性、基於 API 標準的雙向對接能力，使技術底層與 IT 業務流程完美融合。

ITSM 平臺 集成方式 說明

雙向同步能力：ITHub 不僅支持將告警自動創建工單到外部 ITSM 系統，還能將外部工單的狀態變更（受理、掛起、關閉）實時回傳至 ITHub，實現真正的端到端閉環管理。支持 SOAP、REST API 和 Webhook 多種集成方式。
