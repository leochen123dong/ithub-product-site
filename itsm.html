ITIL 标准流程

四大 ITIL 核心流程

ITSM 模块遵循 ITIL 最佳实践设计，覆盖企业 IT 服务管理的核心流程需求。

事件管理 (Incident)

快速恢复服务，最小化业务中断影响。

  • 自动/手动创建工单，告警直接关联
  • P1-P4 优先级分级，SLA 计时器自动启动
  • 拓扑根因分析：多台设备故障合并为一张工单

问题管理 (Problem)

深度分析反复出现的事件，找到并消除根本原因。

  • 事件关联分析，识别反复出现的故障模式
  • 根因分析与解决方案追踪
  • 知识库沉淀，防止同类问题再次发生

变更管理 (Change)

标准化变更流程，降低变更风险，确保所有变更有迹可循。

  • 变更请求审批流程，CI 变更自动触发 CR
  • 变更影响评估，基于 CMDB 拓扑分析
  • 变更实施追踪与回滚计划

服务请求 (Service Request)

标准化处理用户日常服务请求，提升服务效率。

  • 自助服务门户，用户自主提交请求
  • 预定义服务目录和请求模板
  • 请求进度实时查看，服务质量评价
智能告警

智能工单与告警仪表盘

从告警到工单的全自动化闭环，三种工单模式满足不同运营需求。

推荐

全自动模式

基于 Poller 逻辑自动创建和分配工单。

  • 自动创建工单并分配
  • 利用网络拓扑根因分析
  • 多设备故障自动关联合并

半自动模式

控制分配到已有工单，限制新建工单数量。

  • 告警自动关联已有工单
  • 限制重复创建
  • 人工确认后升级处理

手动模式

仅生成告警，用户确认后才创建工单。

  • 告警推送但不自动创建工单
  • 人工评估后决定是否开单
  • 适用于非关键业务场景

SLA 管控

SLA 服务等级管理

基于工单优先级的 SLA 计时器，支持时区感知与自定义覆盖时段，确保服务承诺精准达成。

优先级 说明 响应时间 紧急度
P1 最严重问题，对业务运营产生负面影响且无法通过可接受方式恢复 10 min 紧急
P2 次严重问题，对业务有较大影响但网络仍可运行 20 min 高
P3 一般问题，网络功能降低但对业务影响较小 30 min 中
P4 服务请求或计划性工作 30 min 低
响应与解决分离

响应计时器和解决计时器分别配置，精确追踪服务各环节效率。

时区感知

自动匹配工作时区，自定义工作时间/非工作时间覆盖时段。

SLA 超时预警

SLA 即将超时自动升级告警等级，确保关键问题及时处理。

通知引擎

高度可定制的邮件/短信通知模板，HTML 编辑器支持，事件驱动精准推送。

  • 多语言、多时区感知 — 自动匹配收件人时区和语言
  • 事件驱动触发 — 工单创建/更新/日志创建/SLA 超时
  • HTML 模板编辑器 — 自定义邮件和短信通知格式
  • 第三方集成 — HTTP 调用外部系统，自定义请求头和认证

工单模板与流程

预定义工单模板，支持 HTML 编辑器自定义输入表单，完整的工单生命周期管理。

  • 工单日志 (Journals) — 私密日志、文件上传、草稿
  • 联系人分配 — 自动/手动，区分主要/次要 (Email/CC)
  • 关联工单 — 优化告警路由，避免重复处理
  • 自动关闭缓冲期 — 解决后允许客户在最终关闭前介入

与 CMDB 深度联动

ITSM 与 CMDB 的双向集成是 ITHub 的核心差异化优势

工单自动关联 CI

创建工单时自动关联受影响的配置项（CI），快速定位影响范围。

变更管理集成

资产双向同步

CMDB 中的 CI 资产数据与 ITSM 中的资产记录双向实时同步。

开放生态

开放式生态对接

ITSM 模块绝非孤立系统。ITHub 提供高开放性、基于 API 标准的双向对接能力，使技术底层与 IT 业务流程完美融合。

ITSM 平台 集成方式 说明
Hand 汉得 原生集成 深度适配，支持字段级双向同步和自定义映射
ServiceNow 原生集成 完整生命周期对接，支持工单状态实时同步
Jira Service Management API 集成 通过 REST API 双向对接，自定义字段映射
BMC Remedy API 集成 企业级 ITSM 系统对接
自研系统 开放 API 2,100+ REST APIs 支持任意系统敏捷对接

双向同步能力：ITHub 不仅支持将告警自动创建工单到外部 ITSM 系统，还能将外部工单的状态变更（受理、挂起、关闭）实时回传至 ITHub，实现真正的端到端闭环管理。支持 SOAP、REST API 和 Webhook 多种集成方式。
